# المساهمة في «سطر»

أهلاً بك! «سطر» مشروع مجتمعي مفتوح المصدر، وكل مساهمة مرحَّب بها — من تصحيح خطأ إملائي
إلى ميزة كاملة. هذا الدليل المختصر يشرح كيف تبدأ وما القواعد التي تحافظ على تماسك المشروع.

## التشغيل محلياً

المتطلّبات: **Node.js 18+** و **Claude Code** مثبّتاً عالمياً ومُسجَّل الدخول
(`npm install -g @anthropic-ai/claude-code` ثم `claude`).

```bash
git clone https://github.com/m4u05999/Satr.git
cd Satr
npm install
npm start          # تشغيل للتطوير
npm run dist       # بناء مثبّت ويندوز (يتطلّب «وضع المطوّر» — انظر README)
```

## القواعد المعمارية المثبّتة (لا تُكسَر)

«سطر» يحرص على الأمان والبساطة والعربية أولاً. عند أي مساهمة التزم بالتالي:

1. **الأمان أولاً.** لا تعطّل `contextIsolation` ولا `sandbox`، ولا تفعّل `nodeIntegration`.
   كل قدرة جديدة تُكشف عبر `electron/preload.js` بدالة محدّدة — لا يُكشف `ipcRenderer` كاملاً أبداً.
2. **تحقّق من المدخلات في العملية الرئيسية.** أي قيمة تدخل في وسائط `spawn` تمرّ على تحقّق
   صارم (انظر `SAFE_SESSION` و`SAFE_MODEL` في `electron/main.js`).
3. **العربية أولاً.** كل نص واجهة بالعربية. النصوص المختلطة `dir="auto"`، والأكواد والمسارات
   والأرقام التقنية دائماً `direction: ltr`.
4. **الكود بالإنجليزية، التعليقات بالعربية.** أسماء المتغيّرات والدوال إنجليزية، والشرح عربي.
5. **أقل اعتماديات ممكنة.** الواجهة صفر اعتماديات وقت التشغيل — حافظ على ذلك. العملية الرئيسية
   تعتمد `@anthropic-ai/claude-agent-sdk` فقط.
6. **لا تكسر العقد بين الطبقات.** أي تغيير في صيغة أحداث IPC يتطلّب تحديث الطرفين معاً
   (main ↔ renderer) وتحديث `CLAUDE.md`.
7. **سياسة أمان المحتوى (CSP) صارمة.** لا `'unsafe-inline'`؛ أي `<style>`/`<script>` مضمّن جديد
   في `src/index.html` تُعاد حسبة هاشه تلقائياً عبر `scripts/update-csp.js` (يعمل قبل
   `start`/`dist`). ممنوع السمات المضمّنة (`onclick=…`) — استخدم `addEventListener` و CSSOM.

تفاصيل المعمارية الكاملة وتدفّق البيانات في [`CLAUDE.md`](CLAUDE.md).

## فلسفة العمل: مرحلة واحدة في كل مرة

تطوّر «سطر» عبر **مراحل** موثّقة في [`docs/PLAN.md`](docs/PLAN.md)، بقاعدة ذهبية:
**مرحلة واحدة في كل جلسة عمل**، وبعدها يُشغَّل التطبيق ويُتحقَّق من كل معيار قبول قبل
الانتقال للتالية. حافظ على هذا الإيقاع: اجعل طلبات الدمج (PRs) صغيرة ومركّزة على هدف واحد
واضح، بدل دفعة ضخمة متعدّدة الأغراض.

## خطوات المساهمة

1. اعمل **fork** للمستودع وأنشئ فرعاً وصفيّاً (`fix/diff-undo` مثلاً).
2. طبّق تغييرك مع الالتزام بالقواعد أعلاه.
3. **جرّب يدوياً:** `npm start` وتأكّد أن العربية تُعرض سليمة وأن ميزتك تعمل.
4. افتح **Pull Request** يشرح *ماذا* و*لماذا* بإيجاز. اربطه بأي قضية (issue) ذات صلة.

## الإبلاغ عن العلل واقتراح الميزات

استخدم قوالب القضايا الجاهزة (تظهر عند فتح Issue جديد). للعلل المتعلّقة بعرض العربية،
أرفق **لقطة شاشة** — فهي أوضح من أي وصف.

شكراً لمساهمتك في جعل سطر الأوامر بيتاً عربياً أجمل. 🌙
