## المرحلة 0 — تشغيل الأساس ✅ (جاهزة في هذه الحزمة)

الهيكل الحالي: Electron + محادثة تعمل عبر `claude -p stream-json` + قائمة `/` أساسية + اختيار مجلد أصلي.

**معايير القبول (تحقق منها أولاً قبل أي تطوير):**
- [ ] `npm install` ثم `npm start` يفتح النافذة بدون أخطاء في الـ console
- [ ] إرسال «مرحبا» يعيد رداً عربياً سليم العرض
- [ ] رسالة ثانية تُكمل نفس المحادثة — النموذج يتذكر سياق الرسالة السابقة.
  ملاحظة: معيار القبول هو **استمرارية السياق** لا ثبات المعرّف — بعض إصدارات
  `claude -p --resume` تعيد `session_id` جديداً في كل تشغيل (تفريع جلسة) مع حفظ
  السياق، فتغيّر المعرّف في شريط الحالة ليس خطأ (الواجهة تلتقط المعرّف الجديد
  من حدث `result` وتستأنف به)
- [ ] كتابة `/` تظهر القائمة، والأسهم + Enter تعمل
- [ ] زر 📁 يفتح نافذة اختيار مجلد ويندوز الأصلية
- [ ] زر الإيقاف يوقف طلباً جارياً
- [ ] `npm run dist` ينتج مثبّت `dist/Satr Setup 2.0.0.exe` يعمل بعد التثبيت

## المرحلة 1 — متصفح الجلسات

قراءة جلسات Claude Code المحفوظة محلياً وعرضها واستئنافها.

- المصدر: `~/.claude/projects/` — كل مشروع مجلد باسم مسار مرمّز، بداخله ملفات `.jsonl`
  (كل ملف جلسة؛ افحص بنيتها الفعلية على الجهاز أولاً قبل كتابة المحلل)
- أمر `/جلسات` يفتح لوحة جانبية: عنوان مشتق من أول رسالة مستخدم + التاريخ + المشروع
- النقر على جلسة = تعيين sessionId و cwd ثم استئنافها، مع عرض آخر رسائلها في المحادثة
- IPC جديد: `satr:listSessions` و `satr:readSession` في main.js (قراءة فقط)

**قبول:** أرى جلساتي الحقيقية مرتبة بالأحدث، وأستأنف جلسة قديمة وتكمل من حيث توقفت.

## المرحلة 2 — لوحة الأذونات الحية (الترقية إلى Agent SDK)

النقلة الأهم: استبدال `claude -p` بـ **Claude Agent SDK** في العملية الرئيسية.

- ثبّت حزمة الـ SDK الرسمية (تحقق من اسمها الحالي في التوثيق)
- عند طلب إذن لأداة: مربع حوار عربي في الواجهة (الأداة + التفاصيل + موافقة / رفض / موافقة دائمة)
- بث الرد النصي تدريجياً (token streaming) بدل انتظار اكتمال الرسالة
- زر مقاطعة حقيقي أثناء عمل النموذج
- أبقِ مسار `-p` القديم كاحتياط خلف راية إعداد (fallback) حتى تثبت صلابة المسار الجديد

**قبول:** أطلب مهمة تتطلب تنفيذ أمر نظام في وضع الصلاحيات «افتراضي»، يظهر لي مربع
موافقة عربي، أوافق فيكمل، وأرفض في طلب آخر فيتوقف عن تلك الأداة ويكمل بدونها.

> **🔒 قرار نهائي — سلوك الأذونات (لا يُغيَّر):** المحرك يضبط
> `settingSources: ['user','project','local']`، فيحترم قوائم السماح
> (`allow`) في `settings.json`/`settings.local.json` تماماً كـ Claude Code التفاعلي.
> الأدوات المُصرّح بها مسبقاً تُنفَّذ دون مربع الحوار العربي — هذا **سلوك مقصود ومطابق
> لـ Claude Code، لا خلل**. مربع الحوار يظهر فقط للأدوات غير المُصرّح بها. تم تثبيت هذا
> القرار؛ لا تُعدّله ولا تجعل المربع يعترض كل أداة.

## المرحلة 3 — عارض التعديلات (Diff) العربي ✅ (مكتملة ومقبولة)

- عند أداة Edit/Write: عرض قبل/بعد بألوان (أخضر إضافة، أحمر حذف)، الكود LTR داخل واجهة RTL
- قابل للطي، مع عدّاد أسطر مضافة/محذوفة
- زر «تراجع» يعيد الملف لما قبل التعديل (احتفظ بنسخة المحتوى السابق في الذاكرة)

**قبول:** أطلب تعديل ملف، أرى diff ملوّناً واضحاً، وزر التراجع يعيد الملف فعلياً.

التنفيذ: خطّافا SDK `PreToolUse`/`PostToolUse` يلتقطان قبل/بعد (انظر `electron/agent.js`)،
حساب الفرق في `electron/diff.js` (نقي بلا اعتماديات)، حدث `file_edit` + IPC `satr:undoEdit`.
انظر «عارض الفرق العربي» في CLAUDE.md للعقد الكامل.

## المرحلة 5.5 — تكافؤ الأوامر الأساسية ✅ (مكتملة) ثم **تجميد الأوامر**

دفعة أخيرة مغلقة من ثلاثة أوامر يعتمدها مستخدم Claude Code اليومي، قبل الإطلاق:

- **`/موصلات` (`/mcp`)**: لوحة حالة خوادم MCP (متصل/قيد الاتصال/يحتاج مصادقة/فشل/معطّل)
  مع إعادة اتصال وتفعيل/تعطيل. عبر دوال تحكّم SDK في تشغيل عابر (`withControlQuery`).
  حدّ معروف مقبول: لا مصادقة OAuth داخل التطبيق (يصادَق الموصّل من Claude Code ثم «تحديث»).
- **`/سياق` (`/context`)**: امتلاء نافذة السياق (نسبة + رموز + توزيع الفئات) عبر `getContextUsage()`.
- **`/ضغط` (`/compact`)**: ضغط/تلخيص المحادثة عبر إرسال `/compact` كدور SDK؛ بطاقة قبل/بعد،
  والجلسة تكمل. تحقّق حيّ: `pre_tokens 25528 → post_tokens 1879`، نفس الجلسة، والسياق محفوظ.

التنفيذ: IPC `satr:mcpStatus`/`satr:mcpAction`/`satr:contextUsage` + حدث `compact_boundary`.
انظر «أوامر التكافؤ مع Claude Code» في CLAUDE.md للعقد الكامل.

> **🔒 تجميد الأوامر:** هذه آخر دفعة أوامر قبل الإطلاق. **لا تُضَف أوامر تفاعلية جديدة**
> بعدها (لا hooks ولا subagents ولا أوامر مخصّصة — ميزات متقدمة خارج النطاق). فكرة لوحة
> `/تكلفة` في المرحلة 6 مؤجَّلة احتراماً للتجميد (و`/سياق` يغطّي جزءاً من الحاجة). الانتقال
> التالي للمرحلة 6 (تلميع المنتج) مباشرةً.

**قبول:** `/موصلات` يعرض موصّلاتي بحالاتها وأعيد اتصال واحد؛ `/سياق` يُظهر نسبة الامتلاء
وتوزيع الرموز؛ `/ضغط` يضغط محادثة طويلة وتكمل بالملخّص.

> **⛔ متطلّب حرج — مانع إطلاق (لا غنى عنه):** «سطر» صار يعتمد على Claude Code المثبّت
> عالمياً — محرك SDK يستدعي الثنائي العام عبر `pathToClaudeCodeExecutable` (انظر
> `resolveClaudeBin` في `electron/agent.js`)، ومحرك CLI الاحتياطي يحتاجه أيضاً. لذا
> معالج أول التشغيل **يجب** أن:
> 1. يكتشف غياب Claude Code (نفس منطق `resolveClaudeBin` + فحص الإصدار).
> 2. يرشد المستخدم خطوة بخطوة بالعربية للتثبيت: تثبيت Node ثم
>    `npm install -g @anthropic-ai/claude-code` ثم تسجيل الدخول `claude` مرة واحدة.
> 3. يمنع الوصول للمحادثة قبل توفّره (مع زر «أعد الفحص»).
> هذا **مانع إطلاق وليس تحسيناً اختيارياً** — بدونه يفشل التطبيق صامتاً عند أول رسالة.

**قبول:** مستخدم جديد بلا Claude Code يُرشَد بالعربية لتثبيته خطوة بخطوة، وبعد الفحص
يصل لأول محادثة ناجحة دون فتح طرفية يدوياً.

## المرحلة 7 — إطلاق المصدر المفتوح

«سطر» مشروع مجتمعي مفتوح المصدر برخصة MIT، يُطلَق على GitHub.

- **الرخصة:** ملف `LICENSE` برخصة MIT.
- **README عربي:** شرح المشكلة والحل، خطوات التثبيت، ولقطات شاشة «قبل/بعد» تُظهر
  فرق عرض العربية (RTL + التشكيل) بين الطرفية التقليدية و«سطر».
- **مستودع GitHub:** إعداد المستودع، الوسوم، الإصدارات (Releases)، قوالب القضايا.
- **الرعاية:** زر رعاية متواضع — GitHub Sponsors و/أو `.github/FUNDING.yml`، دون
  إلحاح أو إعلانات.
- **بيان الاستقلالية (إلزامي):** توضيح صريح في README والتطبيق أن «سطر» أداة مجتمعية
  مستقلة، **وليست منتَج Anthropic ولا مدعومة منها رسمياً**؛ Claude Code علامة لـ Anthropic.
- **بيان الخصوصية:** لا تتبّع (no telemetry)، لا جمع بيانات، لا إرسال لأي خادم —
  المحادثات تمر عبر Claude Code المثبّت لدى المستخدم فقط. يُذكر بوضوح في README والتطبيق.

**قبول:** يصل زائر جديد لصفحة GitHub، يفهم ما المشروع من README العربي ولقطاته، يثبّته
بحرّية، ويرى بوضوح أنه مشروع مستقل عن Anthropic بلا أي تتبّع.
